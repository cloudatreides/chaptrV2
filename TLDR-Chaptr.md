## One-liner

**You upload a selfie, become the main character, and live inside an interactive story — with characters who remember you, react to you, and grow attached over time.**

Think *Choose Your Own Adventure* meets *Replika* meets *Studio Ghibli* — but the world reacts to *you* specifically, and the relationships compound across sessions.

---

## The Problem

Three pain points converge on one underserved audience (mostly Gen Z + younger millennials, female-skewing, ~70% of the AI-companion market):

1. **Character.AI / Replika are stuck in a chat box.** Endless text, no progression, no payoff. Users describe "fatigue after 2 weeks."
2. **Visual novels are static.** Pre-written branches, no personalization, the protagonist isn't *you*.
3. **Existing romance/companion apps are gated, mobile-only, and full of celebrity-likeness IP risk.** Simmy, the closest comp, is iOS-only, romance-only, and uses 4 hardcoded choices per beat.

**Chaptr collapses these three into a single web-native experience: a story that adapts to you, with characters that remember you, where the protagonist literally has your face.**

---

## What Chaptr Actually Is

A web app where users:

1. **Create a character** — name, gender, personality, optional selfie. Selfie gets stylized into an anime/illustrated portrait.
2. **Choose a universe** — 15 stories across 6 genres (Romance, Horror, Mystery, Thriller, Fantasy, Adventure).
3. **Read an AI-generated story** — streamed prose, scene images that include the user as the protagonist, dynamic choices generated by Claude (not pre-written).
4. **Build relationships with characters** — affinity score per character, memories, group chats, side quests.
5. **Travel with a companion** — a separate "Travel Mode" where users go on a 5-day AI-generated trip with one of the cast (currently Tokyo, expanding).
6. **Share or come back** — playthroughs persist across devices via cloud sync, with social share URLs and OG cards.

**Core loop time: 1–3 hours per session. Average return rate: target 25%+ within 7 days.**

## Features Built (V2 — Live)

### Story Engine

- **15 universes** across 6 genres, each with multiple chapters and 11+ cast characters
- **Dynamic choice generation** via Claude Haiku — choices adapt to character personality, story state, and trust score
- **Streaming prose** — story arrives word-by-word like a chat, not a wall of text
- **Scene images** — generated via FLUX (Kontext Pro / Schnell) and Gemini Nano Banana, with the user's stylized portrait as the protagonist
- **Trust system** — characters track how much they trust the protagonist; choices shift trust; trust gates outcomes

### Character System

- **Affinity tiers (0–100)** — Stranger → Acquaintance → Friend → Close → Confidant. Affinity gates content (pings, side stories, certain actions).
- **Memory / Callbacks** — characters remember personal facts the user shares mid-conversation. Lightweight extraction call after every 2nd exchange. Memories injected into all future Claude calls per character.
- **Pings** — characters message the user between story beats, unprompted. Affinity-gated.
- **Side quests** — branching mini-stories per character. Quest summaries feed into main story context.
- **Group chats** — multiple characters in one thread, round-robin rotation with AI-to-AI reactions.
- **Free chat mode** — post-story open-ended chat with any character, full playthrough context loaded.

### Chat Actions (Gem Economy)

11 interactive actions across 4 categories — Playful, Gifts, Emotional, Romantic. Each costs gems, gated by affinity tier:

- **Send a Meme** — genre-aware meme pool, character reacts to specific meme
- **Dare** — user dares character; character must perform it
- **Coffee / Serenade** — generates AI scene image of protagonist + character interaction
- **Love Letter** — async AI-generated letter, character reacts emotionally
- **Mystery Box** — random affinity boost
- **Comfort** — character reaction portrait generated

### Travel Mode (GTM Focus)

A separate mode where the user takes a 5-day AI-generated trip with one of the cast as a travel companion. Currently **Tokyo** with companions Sora / Jiwon / Yuna / Kai.

Loop: **Globe → City → Companion + Selfie → Planning Chat → 5-Day Loop (Scene → Chat → Scene → Chat) → Trip Complete**

- Companion has personality sliders (chattiness, planning style, vibe)
- Each day has 3–4 scenes; companion reacts to each in chat
- Day transitions, evening recaps, morning planning
- Trip Complete screen with stats, AI summary, shared highlights, companion farewell
- ~1–2 hours of engagement per trip

### Genre-Aware Story Moments

Selfie capture moments adapt per genre:
- Romance = "Album" (selfie style)
- Thriller = "Dossier" (surveillance aesthetic)
- Horror = "Evidence" (polaroid style)
- Mystery = "Case Files"
- Fantasy = "Memories"
- Adventure = "Journal"

All copy, prompts, and visual styles cascade from a single config — content scales by genre without recoding.

### Auth + Cloud Sync

- Supabase auth + per-user game state sync
- Full store state (chats, affinities, gems, progress, unlocks, trips) persisted to cloud
- Survives logout / login / device switch
- 3 layers of sync resilience: visible status indicator, auto-retry with exponential backoff, server-side error telemetry
- Image cache table prevents regen costs across sessions

### Internal Tooling

- **Image bench** at `/admin/image-bench` — side-by-side A/B testing of 6 image-gen models (FLUX.2 Pro, Kontext Pro, Schnell, Nano Banana 2/Pro/regular). Per-tile latency, cost-per-image, projected $ at 1k images.
- **Image-gen audit** — source-of-truth doc for all 16 image-gen call sites in the app, with one-click load-into-bench for cross-model testing.
- **Sync error telemetry** dashboard via Supabase queries.

## Tech Stack

| Layer | Choice | Why |
|-------|--------|-----|
| Frontend | React + Vite + TypeScript + Tailwind v3 | Fast iteration, type safety, web-native |
| State | Zustand (persist v9, multi-character) | Simple, no Redux overhead |
| Animations | Framer Motion + Vaul | Story-driven, not decorative |

| Image gen | Together AI (FLUX) + Google Gemini (Nano Banana) | A/B testing — Gemini 3.1 Flash currently winning on identity match |
| Backend | Supabase (auth, game state, share URLs, telemetry, image cache) | One service for everything, free tier carries us to 10k MAU |
| Hosting | Vercel (web + Edge functions) | Instant deploy, edge proxies for API keys |

**No native app, no server infra to manage, no team. Solo-buildable end-to-end.**

## Monetisation Plan

**Philosophy: zero paywalls during the experience. Monetisation surfaces only at the highest-emotion moment — when the user is voluntarily asking for more.**

The model mirrors what works in adjacent categories: dating sims monetize "what happens next" cliffhangers; gacha games combine impulse IAP with battle passes; Replika monetizes the bond, not the chat. Chaptr stacks both: an impulse buy at the emotional peak (Phase 1), then a recurring bond-driven product (Phase 2).

### Phase 1: "Stay Longer" — Launch First

**The hook:** the trip-complete screen is the highest-emotion moment in the app. The user just spent 2+ hours building a relationship with their companion, exploring a city, accumulating memories. They don't want it to end.

**The offer:** $2.99 to stay 2 more days with the same companion in the same city. Max 2 extensions per trip (4 extra days, 9 days total).

**Why it works:**
- Impulse buy at peak emotion — highest conversion moment
- Simple to ship — append 2 days to existing state, reuse itinerary infra
- Validates willingness to pay before building anything more complex
- $2.99 is below the "do I really want this?" threshold
- Comparable to dating-sim chapter unlocks ($1.99–4.99) and Chai/Character.AI pay-to-keep-chatting

**Copy:**
- "Not ready to leave? Stay 2 more days with {companion}"
- "Your story doesn't have to end here"
- Frame as invitation from companion, not paywall from app

**Status:** Implemented, currently free for GTM testing. Stripe integration when conversion data justifies it.

### Phase 2: "Return Trip" — Add After Validation

**The retention engine.** Once a user has built a relationship with a companion, they want to come back to that companion — not start over with a stranger.

**The offer:**
- **$4.99 per return trip** — revisit the same city with the same companion. Different itinerary, but companion remembers everything from the first trip.
- **OR $9.99/mo "Companion Pass"** — unlimited returns + priority image gen + exclusive destinations.

**Why it's the long-term play:**
- Every completed trip seeds a future return
- Library of companions × cities grows over time
- Companion memory recall creates emotional lock-in (they remember your jokes, your preferences, your ramen order)
- This is where the real money is in parasocial apps — the long-term bond

**Comparable:** Replika Pro ($49.99/yr), gacha games collecting/revisiting characters.

### Sequencing

| Phase | Purpose | Timing |
|-------|---------|--------|
| Phase 1: Stay Longer | Validate willingness to pay | At launch |
| Phase 2: Return Trip | Build recurring revenue | After 100+ trip completions with Phase 1 data |

**Don't launch both simultaneously.** Two purchase options at trip completion splits attention and kills impulse conversion.

### Pricing Psychology

- **$2.99 anchor** — below "do I really want this?" threshold; price feels trivial vs the 2-hour emotional investment
- **No free extension** — first purchase is the hardest; once they've paid once, the second is easy
- **Show price upfront** — no bait-and-switch, no surprise
- **Last-chance framing** — CTA only appears at trip completion. Scarcity + emotion = conversion

### Future Monetisation Angles (Post-Validation)

- **Premium companions** — new personalities with unique backstories, $2.99 to unlock
- **Premium destinations** — exclusive cities with richer location knowledge, $1.99 to unlock
- **Photo album exports** — shareable trip journal with AI-generated images, $0.99 per export
- **Gift a trip** — send a friend a pre-planned trip with a companion, $4.99

### Solo-Builder Considerations

- Skip subscription infra at launch — Stripe Checkout for one-time purchases only
- Web-only avoids the 30% Apple/Google cut
- Three metrics that matter: **completion rate, extension purchase rate, return rate**

## Why This Wins

1. **Visual + personalized** — no other AI companion app puts the user's actual face into a generated story. Once seen, hard to unsee.
2. **Web-first** — bypasses app-store gatekeepers, the 30% cut, and the iOS-only ceiling that limits Simmy.
3. **Multi-genre** — horror and mystery hit a different audience than romance. One product, three TAMs.
4. **Memory-driven** — characters that remember you across sessions create the parasocial bond Replika monetizes — but inside a story arc, not a chat box.
5. **Travel mode is a Trojan horse** — it's the highest-engagement loop (1–2 hrs per session) and seeds the return-visit retention engine. GTM focus right now.
6. **Solo-buildable** — entire stack runs on free tiers up to ~10k MAU. No team. No fundraise needed to ship.

## What's Open

4. **Stripe integration** — wired in only when Phase 1 conversion data justifies the build cost.
5. **Genre playtesting** — memes, dares, and moments need to be validated across non-romance genres.
